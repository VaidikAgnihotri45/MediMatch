# 🏥 MediMatch — A Healthcare Appointment and Management System for NewLife Hospital

MediMatch is a full-stack Hospital Management System developed using Java, JSP, Servlets, JDBC, and MySQL. The platform is designed to streamline healthcare operations for **NewLife Hospital** by providing appointment scheduling, patient management, doctor workflows, prescription management, OTP-based password recovery, and role-based access control through a centralized system.

## 🚀 Features

### 👤 Patient Module

* Patient Registration and Login
* Secure Authentication System
* Book Doctor Appointments
* View Appointment History
* Manage Personal Profile
* Access Prescription Records

### 👨‍⚕️ Doctor Module

* Doctor Dashboard
* Manage Patient Appointments
* Approve or Reject Appointments
* Generate Digital Prescriptions
* View Patient Medical Information
* Track Consultation History

### 🛠️ Admin Module

* Manage Doctors
* Manage Patients
* Appointment Monitoring
* System Administration
* Hospital Operations Management
* Reports and Controls

### 🔐 Security Features

* OTP-Based Password Recovery
* Email Notification Support
* Session Management
* Role-Based Access Control

### 📋 Prescription Management

* Create and Manage Prescriptions
* Store Medical Records
* View Prescription History
* Digital Prescription Access

## 🔄 System Workflow

1. Patient registers or logs into the system.
2. Patient books an appointment with a doctor.
3. Doctor reviews appointment requests.
4. Doctor approves or rejects appointments.
5. Doctor creates prescriptions after consultation.
6. Patients can access prescriptions and appointment history.
7. Administrators manage users and overall hospital operations.

## ⚙️ Installation & Setup

### Clone Repository

```bash
git clone https://github.com/VaidikAgnihotri45/MediMatch.git
```

### Database Configuration

1. Create a MySQL database.
2. Import the required tables.
3. Update database credentials in the project configuration.

### Run the Project

1. Import the project into Eclipse IDE.
2. Configure Apache Tomcat Server.
3. Deploy and run the application.
4. Access the system through your browser.
